This release makes literature notes safe to keep in sync with your library — a non-destructive update engine that merges reference data with your own edits without ever clobbering your writing — and turns Zotero into a first-class live data source, with two native connection methods, PDF annotations, and tags/collections in templates. It also adds inline citation autocomplete and a references sidebar.

## ✨ New Features

- **Non-destructive literature note updates**: A new sync engine keeps existing notes current without losing your edits. In **Smart sync** (default) the plugin owns only template-rendered frontmatter keys and `{{#syncBlock}}` callouts (tagged with native `^zc-…` block IDs); each unit is **three-way merged** against the last synced snapshot, so library changes and your manual edits combine automatically, and everything else in the note is treated as user content and is never touched. An **Update frontmatter only** mode merges frontmatter and leaves the body alone.
- **Batch and single-note update commands**: **`Citations: Update all literature notes`** re-syncs every note against the current library and content template; **`Citations: Update literature note for current file`** does the same for just the active note. Deleted sync blocks stay deleted, new blocks are appended, and untouched notes are skipped, with a `Updated · Conflicts skipped · Skipped · Errors` summary notice.
- **Conflict review dialog**: When you and the library changed the same thing, the update pauses on a **review dialog** that previews both outcomes as diffs — *Apply (keep my edits)*, *Use library version*, *Skip*, and *Apply/Skip all*. The first sync of a pre-existing note that would append new blocks also asks for consent (opt out via the **Review changes before writing** setting).
- **Native Zotero connection via Better BibTeX**: Load the bibliography directly from a running Zotero over the local Better BibTeX export endpoint — no manual file export. Includes a **Test connection** check, a configurable **Auto-sync interval**, and an offline cache so the library stays usable when Zotero is closed.
- **Native Zotero connection via the local API (no Better BibTeX)**: Connect to the HTTP API built into **Zotero 7+** (`127.0.0.1:23119`) with zero extensions — the most update-resilient way to connect. Resolves citekeys from Zotero's native Citation Key field (or the legacy `Extra` convention), with full metadata, tags, collections, and attachment paths, plus optional group-library and collection scoping.
- **Native Zotero PDF annotations**: With **Import PDF annotations** enabled, the Better BibTeX source fetches real Zotero PDF highlights, underlines, comments, and image/area annotations — each with text, comment, hex color + palette name, page, tags, and a `zotero://open-pdf/...?annotation=KEY` deep link that opens the PDF **at the exact annotation**.
- **Source-agnostic `annotations` template interface**: Zotero PDF annotations and Readwise highlights now share one uniform `annotations` shape (`text`, `comment`, `page`, `pageLabel`, `colorName`, `tags`, `openURI`, `source`), alongside `{{attachments}}` and `{{annotationCount}}` — a single `{{#each annotations}}` template covers both sources.
- **Inline citation autocomplete**: Start typing `@smit` mid-sentence and a popover suggests matching references, inserting your configured citation format in place — no modal needed. Toggle it in **Settings** (on by default).
- **References sidebar**: A side panel (from the ribbon or **`Citations: Show references for current note`**) that lists every reference cited in the active note, resolved live against your library and updating as you switch or edit notes.
- **Zotero tags and collections in templates**: New `{{tags}}` (an alias of `{{keywords}}`) and `{{collections}}` variables expose Zotero terminology directly in your templates.
- **Every data source is a first-class settings entry**: The database configuration UI now presents each source type — file, Zotero (Better BibTeX), Zotero (local API), and Readwise — as a first-class dropdown choice.

## 🐛 Bug Fixes & Improvements

- **Readwise stabilization**: The Readwise source is hardened against transient API failures and API-shape drift, so a flaky response no longer aborts a load.
- **Zotero cache migration**: A legacy-path offline cache is eagerly migrated to the stable-id path, preserving sync continuity across reloads.
- **Sync correctness**: Numerous data-loss paths were closed across the sync/merge subsystem — baselines are durable and flushed on unload, newlines in title-based paths are collapsed, and merges preserve settings and configuration.
