A security and code-quality patch. It updates Handlebars to a fixed release addressing a template-injection advisory, brings the plugin into full compliance with the Obsidian plugin-review lint rules, and raises the minimum Obsidian version to match the APIs it actually uses. There are no user-facing feature changes and no change in runtime behaviour.

## 🔒 Security

- **Handlebars updated to 4.7.9**, resolving a JavaScript-injection-via-AST-type-confusion advisory ([GHSA-3mfm-83xf-c92r](https://github.com/advisories/GHSA-3mfm-83xf-c92r)) that affects 4.0.0–4.7.8. The plugin only ever compiles user-authored templates, so real-world exposure was low, but the fix is a drop-in patch and worth shipping.

## ⚠️ Compatibility

- **Minimum Obsidian version is now 1.7.2** (was 1.4.0). This matches `Workspace.revealLeaf`, the API the references sidebar uses to open its pane; the earlier `1.4.0` floor understated what the plugin needs. Obsidian 1.7.2 shipped in late 2024, so most vaults are unaffected.

## 🔧 Under the Hood

- **Guideline rules enforced locally**: `npm run lint` now reproduces the Obsidian plugin-review findings instead of only catching them in review. `obsidianmd/ui/sentence-case` and `obsidianmd/no-global-this` are errors, product names and acronyms are allow-listed, and `@eslint-community/eslint-comments` forbids silencing a guideline rule or leaving an undocumented `eslint-disable` directive.
- **Suppressions removed**: all five `eslint-disable` comments were deleted and the code they masked was fixed — the Readwise API client now takes an injected timer (so the core layer no longer references `globalThis`), and redundant type assertions were dropped.
- The full test suite (1,709 tests across 93 suites) remains green with 95%+ statement coverage; no runtime behaviour changed.
